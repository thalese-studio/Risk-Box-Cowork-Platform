## 1. 系統定位：物理層防禦節點 (L2 Node) {#定位}

不同於運行在應用層 (L7) 的傳統防火牆，**Risk Box** 運作於數據鏈路層，作為金融系統的「物理保險絲」。

### 1.1 零侵入式旁路部署 (Out-of-Band Deployment)
* **流量鏡像 (TAP/SPAN)**：透過網路分流器獲取 FEP 與 Core 之間的 ISO 電文流。
* **Stealth Mode (隱身運作)**：
    * **NO-IP 配置**：業務監控埠不需配置 IP 位址，在網路拓撲中完全透明，無法被掃描或側向攻擊。
    * **行政零門檻**：不佔用銀行業務 IP 資源，不需修改路由表 (Routing Table) 或 ACL 防火牆規則，大幅縮短上線審核期。
* **偽裝介入 (Spoofed TCP Reset)**：當風險達標，節點偽裝成通訊端點發送 RST 封包。對核心主機而言，此行為僅被識別為客戶端的「正常斷線」。
* **物理 Bypass**：設備故障時採用硬體級電路自動斷開，確保不影響銀行原有營運。

## 2. 實時解析引擎：跨協議行為識別 {#行為DNA}

Risk Box 針對 ISO-8583 與 ISO-20022 關鍵欄位進行正規化解析，建立帳戶行為 DNA。

### 2.1 跨協議電文解析映射表 (Cross-Protocol Mapping)

| 風險監控維度 | ISO-8583 欄位 (L2/L3) | ISO-20022 節點 (L7 XML/JSON) | 偵測邏輯 (Risk Logic) |
| :--- | :--- | :--- | :--- |
| **交易金額** | `DE-4 (Amount)` | `IntrBkSttlmAmt` | 偵測異常的大額或微對稱對沖。 |
| **清算日期** | `DE-7 (Transmission)` | `CreDtTm` | 計算資金停留時長與流轉速度。 |
| **受款帳號** | `DE-102 (Account ID)` | `CdtrAcct/Id/Othr/Id` | 追蹤資金最終落點（如黑名單錢包）。 |
| **商戶類型** | `DE-18 (MCC)` | `Purp/Prtry` | 識別資金流向高風險 VASP 或賭博平台。 |
| **裝置指紋** | `DE-124 (Private)` | `SplmtryData/Envlp` | 透過附加欄位識別跨國異常操作裝置。 |

> **影子轉換技術**：系統採用串流式 SAX 解析技術，確保在混合協定環境下，單筆電文解析延遲 < 5ms。

## 3. 風險演算模型：R-score 權重透明化 {#演算模型}

為符合金融審計與法規遵循，所有介入決策皆基於透明公式：

$$R_{score} = \sum_{i \in DE} (W_i(t) \cdot \delta_i) + \Gamma$$

| 評分維度 | 觸發特徵 | 權重評分 ($\delta$) | 說明 |
| :--- | :--- | :--- | :--- |
| **高齡保護** | 80 歲以上帳戶單筆大額轉帳 | +40 | 若通過 App 面部識別則 -50 |
| **路徑特徵** | 資金流向未曾交易之虛擬貨幣交易所 | +35 | 偵測資產數位化流出的斷點。 |
| **集群特徵** | 相同設備指紋在短時間切換多個帳戶 | +50 | 確定為水房集群作業。 |
| **行為偏離** | 突發性全額資產清空移動 | +45 | 觸發 Ghost Lock 掛起機制。 |
