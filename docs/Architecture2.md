# RISK BOX | 金融通訊協議層聯防架構規格書

> **版本：** v1.2.0 (Stable Release)  
> **技術級別：** L2/L3 協議層即時干預  
> **核心目標：** 透過「物理層阻斷」與「法律避風港」機制，建立秒級斷流防線，解決現有聯防體系不敢即時介入的困局。

---

## 1. 系統定位：物理層防禦節點 (L2 Node) {#定位}

不同於運行在應用層 (L7) 的傳統防火牆，**Risk Box** 運作於數據鏈路層，作為金融系統的「物理保險絲」。

### 1.1 零侵入式旁路部署 (Out-of-Band Deployment)
* **流量鏡像 (TAP/SPAN)**：透過網路分流器獲取 FEP 與 Core 之間的 ISO 電文流。
* **物理介入 (TCP Reset)**：當 $R_{score}$ 達標，節點直接發送 TCP RST 封包強行中斷連線。**核心主機完全不需要修改程式碼**，銀行 CTO 無需承擔系統風險。
* **物理 Bypass**：設備故障時採用硬體級電路自動斷開，確保不影響銀行原有營運。

## 2. 實時解析引擎：行為 DNA 識別 {#行為DNA}

Risk Box 針對 ISO-8583 / ISO-20022 關鍵欄位進行行為權值計算，而非僅依賴靜態黑名單。

### 2.1 關鍵監控數據元 (Data Elements)
* **DE-4 (Amount)**：偵測「微對稱對沖」與異常金額波動。
* **DE-7 (Time)**：計算交易頻次與速率，捕捉極速散金。
* **DE-43 (Location)**：地理空間位移邏輯衝突檢查（如：10 分鐘內跨國操作）。
* **DE-124 (Private Data)**：擷取裝置指紋識別碼，識別水房自動化集群。

## 3. 風險演算模型：R-score 權重透明化 {#演算模型}

為符合金融審計與法規遵循，所有介入決策皆基於透明公式：

$$R_{score} = \sum_{i \in DE} (W_i(t) \cdot \delta_i) + \Gamma$$

| 評分維度 | 觸發特徵 | 權重評分 ($\delta$) | 說明 |
| :--- | :--- | :--- | :--- |
| **高齡保護** | 80 歲以上帳戶單筆大額轉帳 | +40 | 若通過 App 面部識別則 -50 |
| **路徑特徵** | 資金流向未曾交易之虛擬貨幣交易所 | +35 | 偵測資產數位化流出的斷點。 |
| **集群特徵** | 相同設備指紋在短時間切換多個帳戶 | +50 | 確定為水房集群作業。 |
| **行為偏離** | 突發性全額資產清空移動 | +45 | 觸發 Ghost Lock 掛起機制。 |

## 4. 斷流機制：階梯式法律避風港 (The Ghost Lock) {#防誤殺}

這是解決「誤殺客訴」與「法律賠償責任」的核心策略。透過技術手段模擬「通訊故障」，為資產爭取時間。

| 風險等級 | $R_{score}$ | 干預手段 | 客戶感受 | 法律避風港依據 |
| :--- | :--- | :--- | :--- | :--- |
| **Green** | < 60 | 正常轉發 | 正常交易 | 無需介入 |
| **Yellow** | 60 - 80 | **流量抖動 (Jitter)** | 系統稍慢 | 網路波動 (SLA 容許內) |
| **Orange** | 80 - 95 | **Ghost Lock (掛起)** | **連線超時** | **通訊異常 (不可抗力)** |
| **Red** | > 95 | **血液稀釋 (Blood Thinner)**| 交易無效 | 協議一致性檢查失敗 |

> **核心價值：** 銀行在法律上不需為「網路不穩」承擔法律責任，這為警方介入與受害者覺醒創造了 15 分鐘的黃金窗口。

## 5. 聯防機制與隱私保護 {#聯防機制}

* **去中心化共振 ($\Gamma$)**：透過 P2P Gossip 協議同步風險特徵雜湊。
* **隱私聲明**：Risk Box **僅共享行為特徵雜湊值，絕不傳遞任何個資 (PII)**。確保各行數據隱私，消除合規官疑慮。
* **信心度指數 ($C$)**：
  $$C = \frac{\sum (T_j \cdot S_j)}{\sqrt{N_{rep} + \epsilon}}$$
  確保來源節點信任度與分布相似度，防止惡意投毒。

## 6. 落地路線圖：階段性沙盒計畫 {#沙盒計畫}

1. **觀察模式 (Observer Mode)**：旁路部署，僅記錄攔截模擬，驗證誤殺率。
2. **低溫測試 (Jitter Test)**：針對高風險交易執行 3 秒延遲，觀察系統耐受度。
3. **全面啟動 (Active Defense)**：啟用 Ghost Lock 阻斷與跨行情資共振。
