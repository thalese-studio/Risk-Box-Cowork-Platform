# RISK BOX | 金融通訊協議層聯防架構規格書

> **版本：** v1.2.5 (Stable Release)  
> **技術級別：** L2/L3 協議層即時干預 / Stealth Mode  
> **核心目標：** 透過「隱身」與「物理延遲」機制，建立金流延遲防線，解決現有聯防體系因行政與法律責任不敢即時介入的困局。

## 1. 系統定位：物理層防禦節點 (L2 Node) {#定位}

不同於運行在應用層 (L7) 的傳統防火牆，**Risk Box** 運作於數據鏈路層 (L2/L3 協議層)，作為金融系統的「物理緩衝區」。

### 1.1 零侵入式旁路部署 (Out-of-Band Deployment)
* **流量鏡像 (TAP/SPAN)**：透過網路分流器獲取 FEP 與 Core 之間的 ISO 電文流。
* **Stealth Mode (隱身運作)**：
    * **NO-IP 配置**：業務監控埠不需配置 IP 位址，在網路拓撲中完全透明，無法被掃描或側向攻擊。
    * **行政零門檻**：不佔用銀行業務 IP 資源，不需修改路由表 (Routing Table) 或 ACL 防火牆規則，大幅縮短上線審核期。
* **偽裝介入 (Spoofed TCP Reset)**：當風險達標，節點偽裝成通訊端點發送 RST 封包。對核心主機而言，此行為僅被識別為客戶端的「連線異常」。
* **物理 Bypass**：設備故障時採用硬體級電路自動切換，確保不影響銀行原有營運。

## 2. 實時解析引擎：跨協議行為識別 {#行為DNA}

### 2.1 AI 判讀邏輯：行為 DNA 識別 (Behavioral DNA)

Risk Box 除了使用自家銀行內部白名單(靜態規則)，更採用 **RNN (遞歸神經網路)** 過濾電文流向，經由 AI解析 產生行為 DNA：

1. **時間特徵分析**：過濾「快速散金」模式（例如：一筆大額入帳後，數秒內分拆多筆小額 DE-4 轉出）。
2. **拓撲路徑識別**：識別典型的「漏斗型」或「層遞型」資金流動特徵。
3. **動態權重更新**：當特定收單機構 (DE-32) 密集出現遭標記之帳戶時，該路徑之風險權重 $W$ 會即時飆升。

Risk Box 針對 ISO-8583 與 ISO-20022 關鍵欄位進行正規化解析，建立交易行為 DNA。

### 2.2 跨協議 (ISO-8583 & ISO-20022) 解析映射表 (Cross-Protocol Mapping)

| 風險監控維度 | ISO-8583 欄位 (L2/L3) | ISO-20022 節點 (L7 XML/JSON) | 偵測邏輯 (Risk Logic) |
| :--- | :--- | :--- | :--- |
| **交易金額** | `DE-4 (Amount)` | `IntrBkSttlmAmt` | 偵測異常的大額或微對稱對沖。 |
| **清算日期** | `DE-7 (Transmission)` | `CreDtTm` | 計算資金停留時長與流轉速度。 |
| **商戶類型** | `DE-18 (MCC)` | `Purp/Prtry` | 識別資金流向高風險 VASP 或賭博平台。 |
| **收單機構** | `DE-32 (Acq ID)` | `InstgAgt` | **動態權重更新**：當特定路徑密集出現遭標記帳戶時，風險權重即時飆升。 |
| **受款帳號** | `DE-102 (Account ID)` | `CdtrAcct/Id/Othr/Id` | 追蹤資金最終落點（如黑名單錢包）。 |
| **裝置指紋** | `DE-124 (Private)` | `SplmtryData/Envlp` | 透過附加欄位識別跨國異常操作裝置。 |

> **影子轉換技術**：「Risk Box 採用 串流式 XML 解析 (SAX Parser) 解析技術，針對 ISO-20022 僅擷取前 2KB 的關鍵協議標籤，確保在混合協定環境下，單筆電文解析延遲 < 5ms，避免引發銀行核心系統超時。

## 3. 風險演算模型：R-score 權重透明化 {#演算模型}

為符合金融審計與法規遵循，故 風險演算 採用透明公式，如下：

$$R_{score} = \sum_{i \in DE} (W_i(t) \cdot \delta_i) + \Gamma$$


為防止惡意節點進行「數據投毒」，所有協作回報需經過信心度過濾：

$$C = \frac{\sum (T_j \cdot S_j)}{\sqrt{N_{rep} + \epsilon}}$$

* $$T_j$$：來源節點信任等級（如：銀行節點 > 民眾回報）。
* $$S_j$$：特徵空間分布相似度。
* **只有當 $$C > \text{Threshold}$$ 時，該特徵才會正式寫入聯防黑名單。**

| 評分維度 | 觸發特徵 | 權重評分 ($\delta$) | 說明 |
| :--- | :--- | :--- | :--- |
| **高齡保護** | 80 歲以上帳戶單筆大額轉帳 | +40 | 若通過 App 面部識別則 -50 |
| **新戶保護** | 25 歲以下或外籍人士近期開戶/變更紀錄 | +25 | 預警疑似「受誘開立人頭戶」。 |
| **行為偏離** | 滿足 RNN 「快速散金」模型 | +45 | 過濾自動化腳本洗錢行為。 |
| **行為偏離** | 沉睡帳戶 (180天) 突然密集入帳 | +35 | 識別「人頭戶」啟用與測試交易。 |
| **行為偏離** | 突發性全額資產清空移動 | +45 | 觸發 Ghost Lock 掛起機制。 |
| **路徑特徵** | 資金流向高風險 DE-32 機構 | +35 | 針對已知洗錢熱點進行路徑阻斷。 |
| **路徑特徵** | 資金流向未曾交易之虛擬貨幣交易所 | +35 | 過濾資產數位化流出的斷點。 |
| **集群特徵** | 相同設備指紋在短時間切換多個帳戶 | +50 | 疑似水房集群作業。 |
| **物理特徵** | 頻繁更換 VPN 或 IP 地理衝突 | +30 | 識別隱匿行為或受遠端控制特徵。 |
| **聯防標記** | 聯防網標記之黑名單雜湊 ($\Gamma$) | +30 | 實現「一處觸發，全網聯防」。 |

> ### 🔄 維度動態演進機制 (Adaptive Governance)
> 為了應對詐騙洗錢手法的高度變遷性，Risk Box 採用 **「滾動式維度更新計畫」**。本表所列之權重評分 $\delta$ 會根據聯防網之大數據回饋進行動態漂移校準。
> 
> * **即時更新**：針對新出現的惡意 DE-32 機構，權重將在秒級內生效。
> * **不定期擴充**：根據刑事局 (165) 及金融公會發布之新態樣，經由「聯防共識協議」導入新行為維度（如：針對特定族群之社交工程行為偵測）。
> 評分維度會隨風險趨勢滾動更新，但所有權重異動皆具備完整的 數位簽章與審計日誌 (Audit Trail)。任何維度的加入皆須符合個資法與金融監理框架，確保防禦機制的演進既能追上犯罪速度，亦能滿足監理層對演算法透明度的要求。

## 4. 斷流機制：階梯式法律避風港 (The Ghost Lock) {#防誤殺}

這是解決「誤殺客訴」與「法律賠償責任」的核心策略。透過技術手段模擬「通訊故障」，為資產爭取時間。

| 風險等級 | $R_{score}$ | 干預手段 | 客戶感受 | 法律避風港依據 |
| :--- | :--- | :--- | :--- | :--- |
| **Green** | < 60 | 正常轉發 | 正常交易 | 無需介入 |
| **Yellow** | 60 - 80 | **流量抖動 (Jitter)** | 系統稍慢 | 網路波動 (SLA 內) |
| **Orange** | 80 - 95 | **Ghost Lock (掛起)** | **連線超時** | **通訊異常 (不可抗力)** |
| **Red** | > 95 | **血液稀釋 (Blood Thinner)**| 交易無效 | 協議一致性檢查失敗 |

## 5. 聯防機制與隱私保護 {#聯防機制}

* **去中心化共振 ($\Gamma$)**：透過 P2P Gossip 協議同步風險雜湊。
* **隱私保護**：Risk Box **僅共享行為特徵雜湊值，絕不傳遞任何個資 (PII)**。確保各行數據隱私，消除合規官疑慮。

## 6. 落地路線圖：階段性沙盒計畫 {#沙盒計畫}

1. **觀察模式 (Observer Mode)**：Stealth Mode 運作，僅記錄不阻斷，驗證誤殺率 < 0.01%。
2. **微擾測試 (Jitter Test)**：針對高風險交易執行 3 秒延遲，測試系統反應。
3. **全面啟動 (Active Defense)**：啟用 Ghost Lock 阻斷與跨行情資共振。
